# UI 设计说明

## 设计理念

基于专业 UI 工程师的视角，从用户体验和视觉美学角度重新设计界面。

## 核心改进

### 1. 配色方案 (Material Design 3)

- **深色主题**: 更深的背景色层次 (15/25/35)
- **强调色**:
  - 主色 Indigo: `#7877FF` (柔和蓝紫)
  - 辅色 Teal: `#63E6BE` (青绿)
  - 成功: `#4FD17E` (绿色)
  - 错误: `#FF7070` (红色)
- **文字层次**:
  - 主文字: `#FAFAFC` (高对比)
  - 次要文字: `#B4B6C3` (中对比)
  - 弱化文字: `#828491` (低对比)

### 2. 布局优化

- **两列布局**: 左侧配置 + 右侧日志
- **卡片式设计**: 16px 圆角，细边框分隔
- **网格统计**: 2x2 网格展示关键指标
- **合理间距**: 24px 外边距，16-20px 内边距

### 3. 字体层级

```
标题:    32px Bold    (主标题)
副标题:  18px Bold    (卡片标题)
正文:    14px Regular (标签)
数据:    22px Bold    (统计数字)
代码:    13px Mono    (日志)
```

### 4. 交互优化

- **输入框**:
  - 8px 圆角
  - Hover 时边框高亮
  - Focus 时 2px 强调色边框
- **按钮**:
  - 大尺寸主按钮 (48px 高)
  - 圆角 10px
  - 禁用状态灰色
  - Hover 时鼠标指针变化
- **日志滚动**: 自动滚动到底部，粘性底部

### 5. 信息展示

#### 统计卡片 (ProcessStats)

- ✅ 总文件数
- ✅ PDF 发票数
- ✅ 销售方识别率 (百分比)
- ✅ 金额识别率 (百分比)
- ⏱️ 处理耗时 (秒)

#### 彩色日志

- 🟢 成功/完成 - 绿色
- 🔴 错误/失败 - 红色
- 🔵 开始/进行 - 蓝紫色
- ⚪ 信息/数据 - 灰色
- ━━━ 分隔线 - 深灰

### 6. 响应式设计

- 最小窗口: 1000x700px
- 默认窗口: 1100x760px
- 自适应内容宽度
- 滚动区域最大高度 520px

## 用户体验优化

### 操作流程

1. **输入配置** → 清晰的标签和输入框
2. **点击识别** → 大按钮，状态反馈
3. **查看进度** → 实时日志流
4. **查看结果** → 统计卡片自动显示

### 状态反馈

- ⚪ 就绪 - 灰色圆点
- 🟡 处理中 - 黄色圆点 + 按钮禁用
- 🟢 完成 - 绿色圆点 + 统计卡片
- 🔴 失败 - 红色圆点 + 错误信息

### 视觉引导

- 垂直居中标题吸引注意
- 左侧配置 → 操作流程自上而下
- 右侧日志 → 实时反馈，不遮挡
- 底部统计 → 完成后展示成果

## 技术实现

### 组件化

- `render_stat_card()`: 统计卡片组件
- 独立的配色常量
- 分离的布局逻辑

### 性能优化

- 日志限制 200 条（自动清理）
- 条件渲染统计面板
- 最小重绘区域

### 可维护性

- 配色常量集中管理
- 清晰的代码结构
- 注释完整
