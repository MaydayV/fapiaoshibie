这份发票识别工具的设计优化文档旨在提升产品的专业感、操作效率以及信息传达的清晰度。目前的界面功能完整，但在视觉平衡、信息层级和易用性上仍有较大提升空间。

---

# 发票识别工具 UI/UX 优化设计方案

## 1. 核心问题诊断

* **布局破碎**：左侧两个区域（配置、结果）与右侧日志区域比例不协调，视觉重心偏移。
* **信息冗余**：下方的“处理结果”和右侧的“运行日志”在内容上高度重复。
* **易读性差**：处理结果采用垂直文字（直排），不符合中文阅读习惯，识别成本高。
* **交互路径不明确**：配置区、开始按钮、结果反馈的流向不够顺畅。
* **视觉细节**：暗色调对比度需优化，圆角和边距不统一。

---

## 2. 布局优化方案：三段式流式布局

建议将界面调整为更符合工具类逻辑的**“输入 -> 执行 -> 结果”**流程。

### 方案 A：左右分栏（推荐）

* **左侧 (35%)：配置面板**。保持输入项在左侧，优化输入框样式。
* **右侧 (65%)：综合反馈区**。上方为“数据看板（统计指标）”，下方为“详细日志”。
  * *理由*：将静态统计（数字）与动态过程（日志）结合，让用户一眼看到最终成果，同时保留过程回溯。

---

## 3. 具体模块设计规范

### 3.1 配置区 (Configuration)

* **输入框优化**：
  * 取消全圆角，改用 6px-8px 的小圆角。
  * 路径显示：当路径过长时，采用中间省略（...）或仅显示文件夹名，鼠标悬停显示全称。
  * **选择按钮**：将“选择”按钮图标化（文件夹图标）并置于输入框内部右侧，节省空间。
* **开始识别按钮**：
  * 增加渐变色或微阴影，使其在暗色背景下更具点击感。
  * 运行中状态：按钮应显示 Loading 动画，并文字变为“识别中...”。

### 3.2 数据看板 (Processing Results) - 重大调整

* **摒弃垂直文字**：所有文字回归水平排列。
* **卡片式设计**：使用 4 个横向并排的微型卡片显示：
    1. **总文件数** (Total)
    2. **PDF 成功率** (PDF Ratio)
    3. **识别成功率** (Accuracy)
    4. **耗时** (Duration)
* **视觉强化**：使用环形进度条或更有张力的数字大字体（如使用 DIN 字体）。

### 3.3 运行日志 (Execution Log)

* **沉浸式体验**：日志区背景可以比主背景更深一些（如纯黑或深灰），模拟终端感觉。
* **状态图标**：统一图标风格（Outline 或 Solid 选其一），增加不同状态的颜色标识（成功-绿，提示-蓝，错误-红）。
* **输出路径快捷操作**：日志中的“输出文件路径”应支持点击直接打开文件或打开所在文件夹。

---

## 4. 视觉风格指导 (UI Spec)

### 4.1 色彩系统

* **背景色**：`#121212` (主背景), `#1E1E1E` (卡片容器)
* **主色调**：`#00C853` (成功/执行)，建议增加一个品牌辅助色（如科技蓝 `#2979FF`）用于区分配置。
* **文字色**：高亮 `#FFFFFF`, 次要 `#9E9E9E`, 禁用/提示 `#616161`

### 4.2 字体与间距

* **字体**：系统默认黑体 (PingFang SC / Microsoft YaHei)，数字建议使用等宽字体以防数值跳动。
* **间距**：严格遵守 8px 原则（间距均为 8, 16, 24, 32 等倍数），确保视觉呼吸感。

---

## 5. 交互细节建议

1. **拖拽支持**：支持将文件夹直接拖入“发票目录”输入框。
2. **一键打开**：识别完成后，在结果区直接弹出一个“打开 Excel 清单”的显著按钮。
3. **空状态设计**：在未开始识别前，右侧区域不应一片漆黑，可以放置一个简单的插画或操作指南。
